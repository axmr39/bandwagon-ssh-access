## 搬瓦工SSH连接是什么？先搞清楚这三件事

SSH（Secure Shell）是一种用加密方式远程登录服务器的协议。简单说：你在自己电脑上打开一个终端窗口，敲几行命令，就能控制那台放在美国机房里的搬瓦工VPS，就像坐在它面前操作一样。

搬瓦工VPS是完全的root权限、纯Linux环境，没有图形界面，SSH就是你进入它的唯一入口。

要完成一次SSH连接，你需要三样东西：

1. **IP地址**：你的VPS在互联网上的地址
2. **SSH端口**：搬瓦工默认不走22端口，会随机分配一个五位数的端口
3. **root密码**：重装系统时生成的随机密码

这三样东西，都在KiwiVM控制面板里。

## 第二步：根据你的系统，选择连接方式

### Windows系统：用Xshell或PowerShell

**方式一：Xshell（图形界面，新手推荐）**

Xshell是Windows上用得最多的SSH客户端，个人版免费。下载安装后：

1. 打开Xshell，点"新建"创建连接
2. "主机"填入VPS的IP地址
3. "端口"填入KiwiVM里看到的SSH端口号
4. 点"用户身份验证"，方法选"Password"，用户名填`root`，密码填刚才保存的root密码
5. 点"连接"

第一次连接会弹出一个"SSH安全警告"，提示保存主机密钥，点"接受并保存"就可以了。之后再连就不会弹了。

**方式二：PowerShell或Windows Terminal（内置工具）**

Windows 10/11自带SSH客户端，直接打开PowerShell或Windows Terminal，输入：


ssh root@你的IP地址 -p 你的SSH端口


例如：


ssh root@172.80.80.20 -p 28765


回车后输入密码（输入时不会显示任何字符，直接打完按回车就行）。出现`root@bwg:~#`这样的提示符，就说明连上了。

### Mac系统：直接用终端

Mac用户不需要安装额外软件。按`Command + 空格`搜索"终端"打开，或者在应用程序里找到"终端"。

输入命令：


ssh root@你的IP地址 -p 你的SSH端口


例如：


ssh root@172.80.80.20 -p 28765


第一次连接会出现这样一段提示：


The authenticity of host '[172.80.80.20]:28765' can't be established.
Are you sure you want to continue connecting (yes/no)?


输入`yes`回车，然后输入root密码，连接成功。

如果你VPS比较多、经常需要管理，可以试试Termius——Mac和Windows都有，免费版功能足够用，可以保存多台VPS的连接信息，不用每次都手动输命令。

## 连不上怎么办？常见问题排查

### 问题一：连接超时，一直转圈

**可能原因：IP被封了。**

这是搬瓦工用户最常见的问题。某些机场线路质量差，或者服务器跑了某些流量，IP会被GFW封锁。

判断方法：在KiwiVM面板找到"Ping & Traceroute"工具，从国内节点测试，如果ping不通，基本就是IP被封了。

解决办法：在KiwiVM面板里可以申请更换IP（部分套餐免费，部分需要付费），或者直接切换机房。

**也有可能：端口填错了。** 搬瓦工的SSH端口不是22，是五位数的随机端口。回KiwiVM确认一下。

### 问题二：提示"SSH服务器拒绝了密码"

密码输错了。密码是区分大小写的，而且SSH登录时输入密码不会有任何显示，慢慢重输一遍。

如果还是不对，回KiwiVM重装系统生成新密码。

### 问题三：提示"host key verification failed"

这个提示出现在Mac和Linux上，意思是本地保存的主机密钥和服务器现在的密钥对不上了。

通常是你重装过系统，或者换过机房，导致密钥变了。

解决方法：在终端执行以下命令删掉旧的记录（把IP和端口换成你自己的）：


ssh-keygen -R "[你的IP]:你的端口"


然后重新连接，重新保存新密钥。

### 问题四：连上了，但某些命令出现locale警告

Mac连接时可能会出现这行报错：


warning: setlocale: LC_CTYPE: cannot change locale (UTF-8)


不影响使用，是Mac终端把本地的UTF-8编码传过去，而某些Linux系统没有对应配置导致的。如果看着烦，可以在VPS上安装对应的locale包解决。

## 使用SSH密钥登录（进阶，可跳过）

每次登录都输密码确实麻烦。SSH支持用密钥对登录，配置完之后就不用再输密码了。

大致流程：

1. 在本地生成一对密钥（公钥和私钥）：`ssh-keygen -t rsa -b 4096`
2. 把公钥上传到VPS（用`ssh-copy-id`命令，或者手动粘贴到服务器的`~/.ssh/authorized_keys`文件里）
3. 搬瓦工KiwiVM最新版面板支持直接在后台上传SSH公钥，在"SSH Keys"功能里操作更方便

配置好之后，再重装系统时，公钥会自动配置进去，连密码都不用了。

## 常见问题 FAQ

**Q：搬瓦工SSH的默认用户名是什么？**

A：`root`。搬瓦工的VPS给的是完整root权限，直接用root登录就可以，不需要切换用户。

**Q：搬瓦工SSH端口是22吗？**

A：不是。搬瓦工随机分配一个非标准端口，通常是2万到3万之间的五位数。具体端口在KiwiVM面板首页的"SSH Port"位置查看，连接时必须用`-p`参数指定，否则会连接失败。

**Q：SSH密码忘了怎么找回？**

A：KiwiVM面板不提供找回密码的功能。唯一的方式是在"Install new OS"里重装系统，重装完成后页面会显示新生成的随机密码，注意截图或复制保存。

**Q：IP被封了，SSH连不上，怎么临时访问VPS？**

A：KiwiVM面板里有VNC功能（在线SSH）。即使IP被封，VNC走的是另一条通道，依然可以访问控制台进行操作。找到"Launch VNC"按钮，在网页里就能临时登录VPS。

**Q：搬瓦工支持SSH密钥登录吗？**

A：支持。KiwiVM最新版面板已经内置了SSH Keys管理功能，可以直接上传公钥，重装系统时自动配置，非常方便。
